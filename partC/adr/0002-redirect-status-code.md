# ADR-002: Redirect Status Code Сонголт

## Статус
Баталсан

## Огноо
2025-05-12

## Контекст
URL богиносгосон хаяг руу redirect хийхдээ HTTP 301 эсвэл 302 status code ашиглах талаар шийдвэр гаргах шаардлага гарлаа. AI эхлээд 301 санал болгосон.

## Авч үзсэн сонголтууд

### 301 Moved Permanently
- Browser cache хийдэг — хоёр дахь удаагийн дарахад серверт хүсэлт явахгүй
- Click count буруу тоологдоно
- Cache-г арилгахад хэцүү

### 302 Found
- Browser cache хийхгүй — дарах бүрт серверт хүсэлт ирнэ
- Click count зөв тоологдоно
- URL-г хожим өөрчлөх боломжтой хэвээр

## Шийдвэр
302 Found ашиглана.

## Үндэслэл
Click counter бол энэ аппын үндсэн feature-ийн нэг. 301 ашиглавал browser cache-аас шууд redirect хийдэг тул серверт хүсэлт ирэхгүй — click count тоологдохгүй болно. 302 нь энэ асуудлыг шийднэ. URL shortener-ийн олон жишээ (bit.ly г.м) 302 ашигладаг.

## Үр дагавар
- Дарах бүрт серверт хүсэлт ирнэ — server load арай их
- Click count үнэн зөв тоологдоно
- URL-г хожим устгах эсвэл өөрчлөхөд асуудалгүй
